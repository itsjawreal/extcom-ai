## Detailed description

```
Extcom AI Reply drafts reply and quote-tweet options for X/Twitter posts
using your own backend and your own AI provider key (OpenRouter or
OpenAI). Nothing is auto-posted — you review each draft, edit it if you
want, and press Post yourself.

HOW IT WORKS
Click the AI Reply button on any post, pick a tone, and generate up to 3
drafts. Each draft can be copied, regenerated individually, inserted into
the reply composer, or inserted into a Quote Tweet's comment box.

REQUIRES YOUR OWN BACKEND
This extension talks to a small open-source Node.js server that you
deploy yourself (Docker, Railway, Render, Fly.io, or a plain VPS — see the
GitHub repo for one-command setup). Your AI provider API key lives only on
that server, never in the browser. There is no shared service this
extension talks to by default, no signup, and no telemetry — the
developer never sees your posts, drafts, or API usage.

FEATURES
- 24 tone presets, from analytical to deadpan to full degen, or Auto to
  let the AI pick the best fit per post — pin up to 5 as quick-access
  chips
- Manual or Auto reply length, capped at 280 characters
- Optional image reading for posts with attached images (screenshots,
  charts, memes) — up to 4 per post
- Automatic reply-chain context capture, so a reply to a nested comment
  stays on-topic
- Local-only usage history and stats, stored in your browser and nowhere
  else
- A standing default instruction plus a one-off instruction per generation
- Insert into a Reply composer or a Quote Tweet's comment box

Open source (MIT), self-hosted, no SaaS.
Source and setup instructions: https://github.com/itsjawreal/extcom-ai
```

## Privacy tab — permission justifications

**storage**
```
Stores the user's own settings (backend URL, access token, tone/length/
draft-count preferences) and a local-only history of their own
generations, entirely within chrome.storage.local on their device. No
data leaves the device except to the backend URL the user explicitly
configures in Settings.
```

**activeTab**
```
Reads the text, author, and images of the specific X/Twitter post the
user clicks the "AI Reply" button on, so a reply can be generated for
that post. Only accesses the currently active tab, only at the moment the
user clicks the button — no background access to other tabs or browsing
activity.
```

**host_permissions — https://x.com/*, https://twitter.com/***
```
Required to inject the AI Reply button into the post action row on
X/Twitter and to read the content of the post the user clicks it on.
Limited to these two domains.
```

**host_permissions — http://localhost/*, http://127.0.0.1/***
```
Default target for a locally-run instance of the user's own backend
server during development/testing. Not used against any third-party
domain.
```

**optional_host_permissions — http://*/*, https://*/***
```
This extension requires each user to self-host their own backend server,
which can be deployed at any domain the user chooses (their own VPS,
Railway, Render, Fly.io, etc.). This broad optional permission is
requested only when the user enters a custom backend URL in Settings and
clicks Save -- Chrome then shows its own permission prompt scoped to that
specific origin. It is never requested at install time and is never used
for anything other than contacting the single backend URL the user has
configured.
```
